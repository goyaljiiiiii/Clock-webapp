# Contributing

Thanks for considering a contribution to Chronoscape.

## Ways to Contribute
- Report bugs
- Suggest features
- Improve UI/UX or accessibility
- Improve documentation

## Development Setup
1. Fork the repository.
2. Clone your fork.
3. Open `index.html` in a modern browser.

No build tooling is required.

## Guidelines
- Keep changes focused and small.
- Preserve existing code style and naming conventions.
- Use semantic, user-facing commit messages.
- Update documentation when behavior changes.

## Pull Request Checklist
- [ ] Change is scoped and clearly explained.
- [ ] UI works on desktop and mobile.
- [ ] No broken links or missing assets.
- [ ] README is updated (if needed).

## Reporting Issues
When opening an issue, include:
- Expected behavior
- Actual behavior
- Reproduction steps
- Browser and OS details
